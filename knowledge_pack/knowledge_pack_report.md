**生成日期**: 2026-06-20

## 摘要

本知识包为 InsureQuery AI Runtime Kernel 的第一版保险领域知识库。覆盖 **4 大保险类别**、**20 个主流产品**、**30 份监管法规**、**983 个用户问题**。

### 核心结论

> **当前 InsureQuery Runtime 是否已拥有足够的保险知识进入真实世界 Benchmark 测试？**
>
> **答案**: **部分可以，但不完整。** 产品对比、保障范围查询、监管法规检索三类场景已具备基本能力。但 **复杂多跳推理、理赔流程、疾病-产品映射** 三个高频场景覆盖不足。建议按本报告推荐优先级补充数据资产。

## 3. Ontology 缺口分析

### 3.1 当前本体状态

```
EntityType: Product(4), Coverage(6), Disease(5), Regulation(3), Rule(4)
RelationType: contains, covers, defines, implements, regulated_by, excludes, references
总计: 22 entities, 26 relations
```

### 3.2 缺口评估

| 新实体建议 | 优先级 | 理由 |
|---|---|---|
| **InsuranceCompany**（保险公司） | CRITICAL | 20个产品涉及15家公司，当前本体无公司实体 |
| **ClaimCondition**（理赔条件） | HIGH | 高频查询"理赔需要什么条件"无本体支撑 |
| **WaitingPeriod**（等待期） | MEDIUM | 当前作为Rule存在，但缺少产品级链接 |
| **PremiumFactor**（费率因子） | MEDIUM | 年龄、性别、职业等影响保费的因子 |

| 新关系建议 | 优先级 | 理由 |
|---|---|---|
| **belongs_to**（Product→Company） | CRITICAL | 产品须关联保险公司 |
| **requires**（Coverage→ClaimCondition） | HIGH | 保障项链接理赔条件 |
| **applies_to**（Regulation→Product） | MEDIUM | 法规对产品的适用关系 |
| **supersedes**（Regulation→Regulation） | LOW | 法规版本更替 |

### 4.1 摄入缺口

- **条款文档**: 当前仅6份，实际需要 100+ 份

- **FAQ 未摄入**: 983 个 FAQ 尚未结构化摄入到 Evidence Index
